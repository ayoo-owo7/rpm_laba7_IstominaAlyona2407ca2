# Лабораторная работа №7
1. Тема работы: Применение поведенческих паттернов проектирования «Посредник» и «Состояние» при разработке мини-системы принтера. Проектирование очереди печати документов.
2. Цель работы: Формирование практических навыков применения поведенческих
паттернов проектирования в рамках единой программной системы.
3. Теоретическая часть

   Паттерн "Посредник"
Паттерн "Посредник" - один из важных и широко используемых поведенческих шаблонов проектирования, который централизует взаимодействие между объектами через объект-посредник. Вместо того чтобы взаимодействовать напрямую, объекты отправляют сообщения через посредника, что снижает зависимость между ними. Это упрощает управление и обслуживание системы.

  Паттерн "Состояние"
Паттерн "Состояниеп" - это поведенческий паттерн проектирования, который позволяет объекту изменять свое поведение при изменении внутреннего состояния. Он инкапсулирует поведение, зависящее от состояния, в отдельные классы состояний, что позволяет объекту корректно управлять переходами между состояниями. Это позволяет избежать сложных условных операторов и делает систему более удобной в обслуживании и гибкой.
